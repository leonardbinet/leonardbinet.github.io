api_etl.extract_api module¶

Core module using other modules to: - extract data from the Transilien’s API (use utils_api_client) - update it with GTFS data (use query_schedule, if data was previously saved with extract_schedule) - save it in databases, Mongo or/and Dynamo (use utils_dynamo, utils_mongo)

api_etl.extract_api.extract_save_stations(stations_list, dynamo_unique, mongo_unique, mongo_all)¶

This function performs the following operations: - query transilien’s api for the requested stations - transform xml answers into json-serializable items - update items with schedule information - save it in requested databases

Parameters:
  • stations_list (list of str/int) – these are the stations for which the transilien’s api will be queried. List of stations of length 8.
  • dynamo_unique (boolean) – save items in dynamo table that save unique passages
  • mongo_unique (boolean) – save items in dynamo table that save unique passages
  • mongo_all (boolean) – save items in dynamo table that save all passages
api_etl.extract_api.extract_stations(stations_list)¶

This function will query transilien’s API asking for expected trains passages in near future on stations defined as paramater. Note that stations id required by api are in 8 digits format.

This function will return clean data (json serializable), ready to be inserted in Mongo or Dynamo databases.

Parameters:stations_list (list of str/int) – these are the stations for which the transilien’s api will be queried. List of stations of length 8.
Return type:list of dictionnaries (of strings)
api_etl.extract_api.operate_multiple_cycles(station_filter=False, cycle_time_sec=1200, stop_time_sec=3600)¶

Deprecated: a cron job is scheduled to run operate_one_cycle every ‘n’ minutes.

This function performs the operate_one_cycle operation every ‘cycle_time_sec’ seconds (default 10 minutes), during the time defined as ‘stop_time_sec’ (default 60 minutes).

Parameters:
  • station_filter (list of str/int OR False) – default to False. If no filter, the function will take all stations provided by the get_station_ids function. If set to list, these are the stations for which the transilien’s api will be queried. List of stations of length 8.
  • cycle_time_sec (int) – number of seconds between each cycle beginning.
  • stop_time_sec (int) – number of seconds until a new cycle cannot begin.
api_etl.extract_api.operate_one_cycle(station_filter=False, dynamo_unique=True, mongo_unique=False, mongo_all=False)¶

This function performs the extract_save_stations operation in two steps. First half of stations, then second half of stations. This is made so that we do not exceed transilien’s api max requests per minute (350/min).

Parameters:
  • station_filter (list of str/int OR False) – default to False. If no filter, the function will take all stations provided by the get_station_ids function. If set to list, these are the stations for which the transilien’s api will be queried. List of stations of length 8.
  • dynamo_unique (boolean) – save items in dynamo table that save unique passages
  • mongo_unique (boolean) – save items in dynamo table that save unique passages
  • mongo_all (boolean) – save items in dynamo table that save all passages
api_etl.extract_api.save_stations(items_list, dynamo_unique, mongo_unique, mongo_all)¶

This function will save in the specified databases, with uniqueness or not, the items provided as parameter.

Parameters:
  • items_list – the items you want to save. They must be in relevant format, and contain fields that are used as primary fields.
  • dynamo_unique (boolean) – save items in dynamo table that save unique passages
  • mongo_unique (boolean) – save items in dynamo table that save unique passages
  • mongo_all (boolean) – save items in dynamo table that save all passages
api_etl.extract_api.xml_to_json_item_list(xml_string, station, df_format=False)¶

This function transforms transilien’s API XML answers into a list of objects, in a valid format. It might be useful to use a class to define object structure.

Parameters:
  • xml_string (string) – XML string you want to transform
  • station (string/integer) – station id that was queried. This is necessary to add a station field in output. This must be a 8 digits string or integer. You can find the stations ids on SNCF website. https://ressources.data.sncf.com/explore/dataset/referentiel-gares-voyageurs/
  • df_format (boolean) – default False, set True so that the function returns a pandas dataframe instead of a list of dictionnaries.
Return type:

list of dictionnaries of strings only, this is json-serializable so it can easily be inserted in Mongo or Dynamo databases.
